# 작업일지 — 2026-07-12

**작품 링크 갤러리**(mode=`link`) 추가. 학생이 만든 결과물의 주소를 올리면 발표 화면에 카드로 깔리고, 누르면 새 탭에서 열린다.

AI_Pro의 중1 직업체험 수업(2026-07-16, 신한중)에서 학생들이 Canva로 만든 웹사이트를 함께 보려는 요구에서 나왔다.

## 결정

**새 도구를 만들지 않고 기존 세션 구조에 모드를 하나 더 얹었다.**
`meta.mode`가 `word`/`phrase` 두 가지였는데 `link`를 추가했다. 학생 입장에서 참여 흐름(QR → 입력 → 발표 화면)이 워드클라우드와 **완전히 같아서** 새로 배울 게 없다. 강사도 admin 화면 하나만 쓴다.

**집계하지 않는다.** 워드클라우드는 같은 답이 겹쳐야 의미가 있지만 작품 링크는 하나하나가 다르다.
그래서 `expandAliases`·빈도·`hidden`·1표 숨기기를 모두 건너뛰고, 도착한 순서대로 카드에 쌓는다.

**미리보기(썸네일)는 넣지 않았다.** 남의 사이트를 iframe으로 끼워 넣는 건 그 사이트가 허용해야 한다.
Canva가 막아 뒀을 가능성이 있고, 확인하려면 실제 게시된 사이트가 필요하다. 지금은 **별명 + 도메인 카드**로 충분하다.
필요해지면 그때 붙인다.

## 함정 — 보안 규칙의 40자 제한 ★

`entries/$id/w`가 **40자 제한**이었다. Canva 게시 주소는 그보다 길어서 **학생 제출이 전부 거부된다.**
규칙을 모드에 따라 갈라 놓았다.

```
"w": link 모드면 300자 + https:// 로 시작 / 그 외에는 기존 40자
"n": link 모드에서만 허용하는 별명 (12자)
```

**`database.rules.json`을 고치는 것만으로는 적용되지 않는다.** Firebase 콘솔 → Realtime Database → 규칙에
붙여넣고 **게시**를 눌러야 한다. 안 하면 PERMISSION_DENIED.

`http://`는 규칙에서 막았다. 학생이 옛 주소를 붙여넣는 사고를 막고, 수업 화면에서 안전하지 않은 링크를 열지 않기 위해서다.

## 구현

- `admin.html` — 입력 형식에 “작품 링크” 추가. **링크 모드는 1인당 1개로 강제**(제출 개수 선택과 무관)
- `join.html` — 링크 모드면 **별명 칸**이 나타나고 입력창이 URL로 바뀐다. `https://`로 시작하지 않으면
  “브라우저 주소창의 주소를 복사해 붙여넣어 주세요”라고 학생 눈높이로 잡아준다 (중1이 제목·아이디를 붙여넣는 사고가 반드시 난다)
- `present.html` — 링크 모드면 캔버스를 숨기고 `#gallery` 카드 그리드를 띄운다.
  워드클라우드 전용 버튼(1표 숨기기·PNG 저장·일시정지·결과 보기)은 감춘다. 상단 카운터는 “단어 N개” → “작품 N개”
  (텍스트 노드만 바꾼다. `innerHTML`로 갈아 끼우면 `entryCount` 참조가 끊긴다)
- `css/style.css` — `.gallery*` (다크 대응). 캐시 버전 `?v=10`

## 검증

- 세 화면의 인라인 스크립트 문법 검사(`new Function`) 통과, `database.rules.json` JSON 파싱 통과
- 카드 레이아웃은 헤드리스 크롬으로 렌더링해 확인 (1366×768)
- **실제 제출 흐름은 미검증.** Firebase 규칙 게시가 선행돼야 한다
