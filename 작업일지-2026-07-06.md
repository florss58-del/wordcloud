## 함께 반영한 것

### 1. 이모지 → 라인 아이콘
전 화면의 이모지를 **Lucide 계열 인라인 SVG 라인 아이콘**으로 교체.
- 예: 마이크(🎤) → 프레젠테이션 아이콘, 목록/키/클라우드 등
- 발표 툴바 토글 버튼(일시정지·테마·QR)도 JS `ICONS` 상수로 아이콘 처리

### 2. 문장부호 규칙 (한국 실무 문서 기준)
- 단어 연결 `-`, 범위·항목 `–`, 문장 삽입 `—`, 수식 `−`
- **엠 대시는 UI·제목·메뉴·라벨에 사용 금지** → 라벨의 `—`를 가운뎃점(·)으로
- 사용자에게 보이는 따옴표는 둥근따옴표 `“ ”` (곧은따옴표 아님)
- 적용: admin 라벨, 제출 완료 알림, 세션 제거 확인창 등

## 시도했다 접은 방향 (기록)
1. **컬러팝**(인디고–바이올렛–핑크 그라디언트·메시 배경) → "너무 AI스럽다"
2. **파티클 텍스트 효과**(21st.dev) 배너 → "정신없다"
3. **WebGL 셰이더 히어로 배경** → 화면 전체가 어두워 어색 + 일부 환경서 회색 빈 화면
4. **Apple 극단 미니멀** → "디자인이 하나도 안 된 것 같다"(밋밋)
5. **Linear 다크 프리미엄** → "너무 까매서 글이 안 보인다"
6. → **Apple 폴리시드(현재)로 확정**

> 교훈: 화려한 효과(그라디언트·파티클·셰이더)는 지양. 화면 주인공은 항상 "단어". 절제하되 여백·타이포·은은한 깊이로 고급감을 낸다.
> React/shadcn 컴포넌트는 정적 HTML 구조라 canvas 로직만 바닐라 JS로 포팅해 실험(모두 최종 미채택).
