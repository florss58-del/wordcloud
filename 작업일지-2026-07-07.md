# 작업일지 — 2026-07-07

강사 페이지 잠금(TODO 항목)을 **Firebase Authentication + RTDB 보안 규칙**으로 구현.

## 왜 이 방식인가

- 정적 사이트라 HTML/JS에 심는 비밀번호는 소스에서 보이는 "가림막"일 뿐이고, DB가 열려 있으면 페이지를 우회해 API로 직접 쓰기 가능
- 진짜 잠금은 Firebase 쪽에서만 가능 → 이메일/비밀번호 로그인 + 규칙으로 쓰기 권한 제한
- 메인 도메인(Y:\claude\test)의 게이트(gate.js, SHA-256 클라이언트 측)에 링크를 숨기는 방안도 검토했으나, 링크만 가려질 뿐 워드클라우드 주소·DB는 그대로라 대체 불가 판단

## 구현 내용

- `js/auth.js` (신규): `requireInstructor(onReady)` — 미로그인 시 전체 화면 로그인 오버레이, 로그인 후 콜백 실행. 마지막 이메일 기억, 한국어 오류 메시지, `signOutInstructor()`
- `admin.html`: 로그인 게이트 적용 + 푸터에 로그아웃 링크
- `present.html`: 로그인 게이트 적용 (`requireInstructor(init)`) — 단어 숨김·초기화가 강사 권한이므로 발표 화면도 게이트
- `join.html`·`index.html`: 변경 없음(학생용, 인증 불필요). CSS 캐시 버전만 `?v=5` 통일
- `css/style.css`: `.auth-gate`/`.auth-card` 등 게이트 스타일 추가
- `database.rules.json` (신규, 규칙 원본):
  - `meta` 생성: 로그인 필수 + 생성 후 수정·삭제 불가 유지
  - `entries` push: 익명 허용(학생), 단 세션 존재 시에만 + `w` ≤ 40자 검증
  - `entries` 전체 삭제(초기화)·`hidden` 쓰기: 로그인 필수
  - 세션 목록 열거(`/sessions` 읽기) 차단, 코드 단위 읽기만 허용
